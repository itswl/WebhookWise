# 如何查看 AI 分析和原始数据

## 新功能：智能按需加载 🚀

为了提升页面加载速度，我们优化了数据加载策略：
- **列表加载**：使用轻量级摘要数据（速度提升20倍+）
- **详情查看**：首次展开时自动加载完整数据

## 使用方法

### 方法1：点击展开告警项（推荐）⭐

1. **打开 Web 界面**
   ```
   访问 http://localhost:8000
   ```

2. **点击任意告警项**
   - 点击告警的标题栏（不要点击按钮）
   - 告警项会展开，显示详细信息

3. **自动加载完整数据**
   - 首次展开时，系统会自动调用 `/api/webhooks/<id>` 接口
   - 加载完整的原始数据和 AI 分析结果
   - 显示加载动画，几十毫秒后完成

4. **切换标签页查看**
   - **概览**：基本信息（ID、来源、时间等）
   - **原始数据**：完整的 JSON 数据
   - **AI 分析**：完整的智能分析结果

### 方法2：直接调用 API

#### 查看单条完整数据
```bash
# 获取 ID=8265 的完整数据
curl http://localhost:8000/api/webhooks/8265 | jq .

# 响应示例
{
  "success": true,
  "data": {
    "id": 8265,
    "source": "mongodb",
    "timestamp": "2026-02-27T17:50:00",
    "raw_payload": { ... },        # 完整原始数据
    "parsed_data": { ... },        # 解析后的数据
    "ai_analysis": {               # 完整 AI 分析
      "summary": "MongoDB主机连接数异常...",
      "event_type": "资源耗尽",
      "impact_scope": "单个MongoDB实例",
      "actions": ["立即检查连接池配置", "..."],
      ...
    },
    "importance": "high",
    ...
  }
}
```

#### 加载列表（摘要模式，默认）
```bash

curl "http://localhost:8000/api/webhooks?page=1&page_size=100" | jq .

# 响应字段（仅11个）

    {
      "id": 8265,

```

#### 加载列表（完整模式）
```bash

curl "http://localhost:8000/api/webhooks?page=1&page_size=50&fields=full" | jq .

# 响应字段（16个，包含 raw_payload, ai_analysis 等）
```

## 性能说明

### 为什么要分阶段加载？

**优化前**（加载5000条请求）：
- 响应时间：~800ms
- 数据传输：~1.5 MB
- 用户体验：卡顿 ❌

**优化后**：
- **初次加载列表**：
  - 响应时间：~40ms（提升20倍）⚡
  - 数据传输：~78 KB（减少95%）⬇️
  - 用户体验：流畅 ✅

- **展开查看详情**：
  - 响应时间：~20ms（单条）
  - 数据传输：~15 KB（单条）
  - 用户体验：即点即现 ✅

### 加载过程可视化

```
用户访问页面
    ↓
加载 200 条摘要数据（40ms，78KB）
    ↓
显示告警列表 ✅
    ↓
用户点击展开某条告警
    ↓
自动加载该条完整数据（20ms，15KB）
    ↓
显示完整 AI 分析和原始数据 ✅
```

**总体验**：初次加载快 + 按需加载详情 = 最佳用户体验 🎉

## 常见问题

### Q1: 为什么第一次展开会有短暂加载？
**A**: 为了提升页面初始加载速度，完整数据采用按需加载策略。首次展开时加载，后续再次展开直接从缓存读取。

### Q2: 如何判断数据已经加载完成？
**A**:
- 初次加载：看到"正在加载完整数据..."的提示
- 加载完成：看到格式化的 JSON 数据和完整的 AI 分析
- 浏览器控制台会显示：✅ 完整数据加载成功

### Q3: 可以强制刷新某条告警的数据吗？
**A**: 可以使用"重新分析"按钮，这会重新调用 AI 分析并更新数据。

### Q4: 导出数据时会包含完整信息吗？
**A**: 如果需要导出完整数据，建议使用：
```bash
# 分批导出完整数据
curl "http://localhost:8000/api/webhooks?page=1&page_size=200&fields=full" > export_page1.json
curl "http://localhost:8000/api/webhooks?page=2&page_size=200&fields=full" > export_page2.json
# ...
```

或者通过游标分页：
```bash
curl "http://localhost:8000/api/webhooks?page=1&page_size=200&fields=full" > page1.json
# 从 page1.json 获取 next_cursor
curl "http://localhost:8000/api/webhooks?cursor=8147&page_size=200&fields=full" > page2.json
```
