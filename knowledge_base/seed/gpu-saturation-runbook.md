# GPU 资源饱和告警处置预案

## 适用告警
- 指标 `GpuUsedUtilization`（GPU 使用率）≥ 90%
- 指标 `GpuMemoryUsedUtilization`（GPU 显存使用率）≥ 95%
- 常见资源：comfyui 模型推理节点（如 comfyui-model02-n01）、训练任务实例

## 这是什么
GPU 计算服务承载图像/模型推理与训练任务。使用率或显存持续打满会导致推理排队、超时，甚至 OOM 杀进程。短时打满（分钟级尖峰）在批量推理场景属正常，**持续 15 分钟以上的高位**才需要人工介入。

## 处置步骤
1. 确认是单卡还是整机：查看告警维度里的 `resource_id`（如 i-xxxx）定位实例。
2. 区分场景：
   - **批量推理高峰**：通常可自愈，观察是否在任务队列清空后回落。
   - **显存泄漏/僵尸进程**：使用率低但显存不降，需重启对应推理进程。
3. 扩容：如持续打满且队列堆积，联系算力平台组 @infra-gpu 临时扩容推理副本或迁移任务。
4. 限流：必要时在推理网关侧对低优任务降级/排队。

## 负责人与升级路径
- 一线：算力平台组 @infra-gpu
- 升级：P0（核心业务推理受阻）直接电话值班，P1 工作时间内处理。

## 备注
WebhookWise 对 GPU 饱和类告警有确定性规则会把重要性提升到 high（即使 AI 判低）；该类告警默认会被转发，除非命中静默。
