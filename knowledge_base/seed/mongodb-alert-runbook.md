# MongoDB 告警排查预案

## 适用告警
- 连接数高 / 连接池打满
- 主从复制延迟（replication lag）
- 慢查询、锁等待、磁盘使用率高
- 来源：云监控 MongoDB 实例告警

## 这是什么
MongoDB 承载业务文档存储。常见告警按根因分三类：连接、复制、资源。处置前先确定是哪一类，避免盲目重启（重启主节点会触发选主，放大影响）。

## 处置步骤
1. **连接数高**：
   - 查是否有连接泄漏（应用未释放连接 / 连接池配置过大）。
   - 临时缓解：调小应用侧连接池上限；切勿直接重启实例。
2. **复制延迟**：
   - 查从节点负载、网络、oplog 是否够大。
   - 大批量写入导致的延迟通常随写入结束回落。
3. **慢查询/锁**：
   - 用 `db.currentOp()` 看长事务/锁等待，必要时 kill 单个失控操作（不要重启实例）。
   - 检查是否缺索引导致全表扫描。
4. **磁盘高**：清理日志/历史集合或扩容卷。

## 红线
- **禁止**未经 DBA 确认直接重启主节点或 step down——会触发选主、连接全断、业务抖动。
- 写操作（kill op、改配置）走 DBA 审批。
