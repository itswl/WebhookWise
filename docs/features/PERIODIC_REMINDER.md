# 周期性提醒功能

> 版本：v1.0
> 更新时间：2026-02-28
> 状态：✅ 已实现

---

## 📖 功能概述

周期性提醒功能确保**长时间持续的告警不会被遗忘**。即使告警在24小时窗口内，如果持续时间过长，系统也会周期性地发送提醒通知。

### 问题场景

**修复前的问题**：
```
第0小时：告警来了 → 通知 ✓
第1-23小时：持续重复 → 静默（窗口内重复）
第24小时：窗口外重复 → 通知 ✓

问题：告警持续了24小时，但只通知了2次！
```

**修复后的行为**：
```
第0小时：告警来了 → 通知 ✓
第6小时：持续重复 → 周期性提醒 🔔
第12小时：持续重复 → 周期性提醒 🔔
第18小时：持续重复 → 周期性提醒 🔔
第24小时：窗口外重复 → 通知 ✓

优势：每6小时提醒一次，不会忘记持续的问题！
```

## ⚙️ 配置说明

### 环境变量配置

在 `.env` 文件中设置：

```bash
# 是否启用周期性提醒（推荐启用）
ENABLE_PERIODIC_REMINDER=true

# 提醒间隔（小时，推荐 6）
REMINDER_INTERVAL_HOURS=6
```

### 配置参数详解

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `ENABLE_PERIODIC_REMINDER` | boolean | `true` | 是否启用周期性提醒 |
| `REMINDER_INTERVAL_HOURS` | integer | `6` | 提醒间隔（小时），范围 1-24 |

## 🎯 工作原理

### 核心逻辑

```python
if 告警是窗口内重复:
    if 距离上次通知时间 >= REMINDER_INTERVAL_HOURS:
        发送周期性提醒
        更新 last_notified_at = 当前时间
    else:
        跳过通知
```

### 数据库字段

新增字段：`last_notified_at TIMESTAMP`

- **新告警**：`last_notified_at` 设置为创建时间
- **周期性提醒**：每次成功转发后更新
- **窗口外告警**：重新通知时更新

### 飞书通知示例

**周期性提醒的消息标题**：
```
🔔 周期性提醒：告警持续中（已重复 42 次）
```

**普通通知的消息标题**：
```
📡 Webhook 事件通知
```

## 🔄 与其他功能的配合

### 与窗口外告警配合

```bash
# 推荐配置组合
DUPLICATE_ALERT_TIME_WINDOW=24       # 24小时窗口
FORWARD_AFTER_TIME_WINDOW=true       # 窗口外告警转发
ENABLE_PERIODIC_REMINDER=true        # 启用周期性提醒
REMINDER_INTERVAL_HOURS=6            # 每6小时提醒

# 场景演示
第0小时：新告警 → 通知 ✓
第6小时：窗口内重复 → 周期性提醒 🔔
第12小时：窗口内重复 → 周期性提醒 🔔
第18小时：窗口内重复 → 周期性提醒 🔔
第24小时：窗口外重复 → 窗口外通知 ✓（重新开始计时）
第30小时：窗口内重复 → 周期性提醒 🔔
```

### 与混合窗口逻辑配合

系统使用**混合窗口逻辑**，防止持续告警一直在窗口内：

```python
# 窗口起点选择
if 有最近的 beyond_window=1 告警:
    窗口起点 = 最近的窗口外告警时间
else:
    窗口起点 = 原始告警时间

# 判断
if 距离窗口起点 > 24小时:
    beyond_window = 1  # 窗口外，触发新一轮通知
```

**双重保障**：
1. **混合窗口逻辑**：超过24小时自动变为窗口外
2. **周期性提醒**：窗口内每6小时提醒一次

## ❓ 常见问题

### Q1: 周期性提醒会影响性能吗？

**答**：不会。只在转发时检查一次 `last_notified_at`，性能开销可忽略不计。

### Q2: 如果告警在窗口内自行恢复了，还会提醒吗？

**答**：不会。只有告警**持续重复**才会触发提醒。如果告警停止，不会再有新的重复记录，自然也不会提醒。

### Q3: 可以为不同的告警设置不同的提醒间隔吗？

**答**：当前版本是全局配置。未来可以考虑支持基于告警类型、重要性等维度的差异化配置。

### Q4: 周期性提醒会发送给谁？

**答**：发送到 `FORWARD_URL` 配置的目标（通常是飞书群组）。与普通告警的接收人相同。

### Q5: 关闭周期性提醒后，历史数据怎么办？

**答**：关闭后不影响历史数据，只是不再触发新的周期性提醒。`last_notified_at` 字段仍然保留。
